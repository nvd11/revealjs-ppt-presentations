What is an LLM?

🧠

An LLM is like a "Digital Brain" with massive knowledge

✅ What it CAN do

  • Answer your questions
  • Provide suggestions & steps to solve problems
  • Generate plans and methodologies

❌ Its Core Limitation

  • It's a "brain in a vat" with no hands
  • It can only "think" and "speak"
  • It CANNOT "execute" any specific actions for you!

What is an Agent?

🦾 🧍‍♂️

An Agent is the "Body and Hands"

🧠 + 🦾 Inseparable from the LLM

A body cannot act without a brain. The Agent relies entirely on the LLM for reasoning, planning, and decision-making.

⚙️ Format, Control & Execute

  • Understands the LLM's intent and output
  • Standardizes & Controls the LLM's response
  • Executes actions using external tools on behalf of the brain

Building a SqlAgent 💻

  • 1. Build Tool
    Python func execute_sql(sql) returns DB data.
  • 2. System Prompt (JSON)
    Instruct LLM to output specific JSON if tools are needed.
  • 3. Call LLM
    Send prompt + user query in a loop.
  • 4. Check Intent (Parse JSON)
    Parse JSON to check if tool key exists.
  • 5. Execute & Loop
    Run tool, append result, loop.
  • 6. Final Answer
    If not JSON or no tool -> exit loop.
import json
class SqlAgent:
    # 1. Build a Tool
    def execute_sql(self, sql_string: str):
        return db.query(sql_string)

    # 2. System Prompt
    sys_prompt = """You are a SQL Agent. 
    Tool: execute_sql(sql_string).
    To use a tool, you MUST output JSON: 
    {"tool": "execute_sql", "args": {"sql_string": "SELECT..."}}
    Otherwise, output final answer in plain text."""

    def run(self, user_query: str):
        messages = [self.sys_prompt, user_query]
        while True:
            # 3. Call LLM
            res_text = llm.chat(messages)

            # 4 & 6. Parse JSON to Check Intent
            try:
                action = json.loads(res_text)
                if "tool" in action and "args" in action:
                    # 4. Intent Yes -> Execute Tool
                    sql = action["args"]["sql_string"]
                    tool_data = self.execute_sql(sql)

                    # 5. Append Response & Loop
                    messages.append(res_text)
                    messages.append(f"Observation: {tool_data}")
                    continue
            except json.JSONDecodeError:
                pass

            # No JSON / No Tool -> Return final answer
            return res_text

What is MCP? 🔌

(Model Context Protocol: Open Standard for Tools)

🧱 Local Function Tool

🤖 -- calls --> ⚙️
  • Hardcoded in the Agent's codebase
  • Bound to the same language (e.g., Python agent -> Python tool)
  • Adding new tools requires code changes

🌐 MCP Server & Tool

🤖 <-- stdio/SSE --> 📦
  • Runs as a standalone process (Server)
  • Language agnostic (Python agent can call TS tool)
  • Plug & Play: Agent dynamically discovers available tools!

Code Comparison: Local vs MCP

🧱 Local Tool (Hardcoded)

# 1. Define function directly in Python
def fetch_weather(city: str):
    return weather_api.get(city)


# 2. Hardcode the execution routing
if tool_name == "fetch_weather":
    result = fetch_weather(args["city"])

🌐 MCP Tool (Dynamic)

# 1. Connect to standalone Server (e.g. Node.js)
client = MCPClient(cmd="npx", args=["-y", "weather-mcp"])

# 2. Auto-discover tools dynamically!
# No need to rewrite system prompts manually
tools = await client.list_tools() 

# 3. Call tool via standard protocol (JSON-RPC)
result = await client.call_tool(
    tool_name, tool_args
)
Why it's a Game Changer 🚀:
Instead of writing/importing every tool yourself, you just start a community MCP Server (like GitHub, Slack, or File System MCP) and connect your Agent to it. The Agent automatically discovers the tools and gains superpowers instantly!
